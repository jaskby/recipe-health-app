## 一、产品概述

### 1.1 产品定位

食谱健康APP是一款集食材价格查询、智能食谱推荐、减肥计划定制、体重管理于一体的综合性健康饮食管理应用。通过整合本地食材价格数据，结合用户健康指标，为用户提供经济实惠、营养均衡的个性化饮食方案。

### 1.2 目标用户

- 关注健康饮食的普通用户
- 有减肥需求的用户
- 注重食材性价比的家庭主妇/主夫
- 追求便捷饮食推荐的上班族

### 1.3 核心价值

- **经济实惠**：实时本地食材价格，帮助用户合理规划饮食预算
- **健康科学**：基于营养学的个性化食谱推荐
- **便捷高效**：一键生成食谱，省去每日纠结"吃什么"的烦恼
- **目标导向**：减肥计划可视化，帮助用户达成健康目标

## 二、功能模块详细设计

### 2.1 每日食材价格更新功能

#### 2.1.1 功能描述

系统每日自动更新用户所在地区的超市和市场食材价格信息，涵盖蔬菜、肉类、海鲜、水果等常见食材类别。

#### 2.1.2 功能需求

| 需求项 | 详细说明 |
|--------|----------|
| 数据来源 | 定位当地超市/市场（如：沃尔玛、永辉、菜市场等） |
| 更新频率 | 每日更新一次，更新时间为每日凌晨 |
| 食材分类 | 蔬菜类、肉类、海鲜类、水果类、蛋奶类、粮油类、调味品类 |
| 价格展示 | 单价（元/斤或元/个）、当日涨跌幅、近7日价格趋势图 |
| 定位方式 | GPS自动定位 + 手动选择城市/区域 |

#### 2.1.3 数据字段

```
食材信息表：
- 食材ID
- 食材名称
- 食材分类
- 食材图片
- 当前价格
- 价格单位
- 昨日价格
- 涨跌幅
- 更新时间
- 数据来源（超市/市场名称）
- 营养成分（热量、蛋白质、脂肪、碳水化合物等）
```

#### 2.1.4 界面设计要点

- 首页展示价格波动较大的食材（涨跌TOP10）
- 支持按分类筛选查看
- 支持搜索特定食材
- 支持收藏关注特定食材价格变化
- 价格趋势图表展示（折线图）

### 2.2 每日推荐一日三餐功能

#### 2.2.1 功能描述

根据用户的身高、体重、年龄、性别等健康指标，结合用户设置的口味偏好，为用户智能推荐营养均衡的一日三餐食谱。

#### 2.2.2 用户健康信息采集

| 信息项 | 是否必填 | 说明 |
|--------|----------|------|
| 身高 | 可选 | 用于计算BMI和每日所需热量 |
| 体重 | 可选 | 用于计算BMI和每日所需热量 |
| 年龄 | 可选 | 影响热量计算公式 |
| 性别 | 可选 | 影响热量计算公式 |
| 活动量级别 | 可选 | 久坐/轻度活动/中度活动/重度活动 |

> 注：若用户不设置健康信息，系统按照普通成年人标准（男性2000kcal/女性1800kcal）随机生成食谱。

#### 2.2.3 口味偏好设置

| 偏好类型 | 选项 |
|----------|------|
| 口味偏好 | 清淡、适中、重口味、麻辣、酸甜、咸鲜等 |
| 忌口食材 | 用户自定义添加不吃的食材 |
| 过敏食材 | 用户自定义添加过敏食材 |
| 饮食习惯 | 正常、素食、清真、低碳水等 |
| 烹饪方式偏好 | 炒、蒸、煮、烤、凉拌等 |

#### 2.2.4 食谱生成规则

```
营养配比原则：
- 碳水化合物：占总热量50-60%
- 蛋白质：占总热量15-20%
- 脂肪：占总热量20-30%

三餐热量分配：
- 早餐：占总热量25-30%
- 午餐：占总热量35-40%
- 晚餐：占总热量30-35%

食材搭配原则：
- 每餐包含：主食 + 蛋白质 + 蔬菜
- 食材种类多样化，每日不少于12种食材
- 颜色搭配丰富
```

#### 2.2.5 食谱输出内容

| 内容项 | 说明 |
|--------|------|
| 菜品名称 | 早餐/午餐/晚餐各菜品名称 |
| 菜品图片 | 高清成品图 |
| 食材清单 | 所需食材种类及数量（精确到克） |
| 做法步骤 | 详细烹饪步骤说明 |
| 营养成分 | 每餐热量、蛋白质、脂肪、碳水含量 |
| 预估成本 | 基于当日食材价格计算 |
| 烹饪时长 | 预估烹饪所需时间 |
| 难度等级 | 简单/中等/困难 |

#### 2.2.6 输出形式

- 以图片形式输出一日三餐完整食谱
- 图片包含：菜品图片、食材清单、做法步骤、营养信息
- 支持保存图片到相册
- 支持分享到社交平台

### 2.3 减肥计划食谱功能

#### 2.3.1 功能描述

为有减肥需求的用户提供定制化减肥食谱计划，根据用户设定的减重目标，科学规划减重周期和每日饮食方案。

#### 2.3.2 减肥计划设置流程

```
Step 1: 输入当前体重
Step 2: 输入目标体重
Step 3: 系统计算减重千克数
Step 4: 系统推荐减重周期（可选调整）
Step 5: 生成减肥计划食谱
Step 6: 用户确认并收藏
```

#### 2.3.3 减肥计划计算逻辑

```
减重速度建议：
- 健康减重速度：每周0.5-1kg
- 减重周期（天）= 减重目标(kg) ÷ 0.5(kg/周) × 7天

热量缺口计算：
- 每日热量需求 = BMR × 活动系数
- 减肥期每日热量 = 每日热量需求 - 500kcal（热量缺口）
- 最低不低于1200kcal（女性）/1500kcal（男性）

BMR计算公式：
- 男性：BMR = 88.362 + (13.397 × 体重kg) + (4.799 × 身高cm) - (5.677 × 年龄)
- 女性：BMR = 447.593 + (9.247 × 体重kg) + (3.098 × 身高cm) - (4.330 × 年龄)
```

#### 2.3.4 减肥计划内容

| 内容项 | 说明 |
|--------|------|
| 减重目标 | 目标减重千克数 |
| 计划周期 | 达成目标所需天数 |
| 开始日期 | 计划开始日期 |
| 结束日期 | 预计达成日期 |
| 每日食谱 | 减重期间每日三餐详细食谱 |
| 每日价格 | 基于当日食材价格计算的单日食谱成本 |
| 总计花费 | 减重期所有食谱的总预估花费 |
| 进度追踪 | 已完成天数/总天数 |

#### 2.3.5 减肥计划食谱特点

- 热量控制在合理范围，保证营养均衡
- 高蛋白、低脂肪、适量碳水
- 增加膳食纤维摄入
- 避免高热量、高油脂食材
- 提供饱腹感强的食材选择

#### 2.3.6 用户主页减肥计划展示

```
我的减肥计划页面：
- 进行中的计划（高亮展示）
- 已完成的计划（历史记录）
- 计划详情页：
  - 计划概览（目标、进度、花费）
  - 日历视图（每日食谱）
  - 今日食谱（快速查看）
  - 体重变化曲线
  - 取消/暂停/继续计划
```

### 2.4 每日体重变化记录功能

#### 2.4.1 功能描述

用户可每日或定期记录体重数据，系统生成体重变化趋势图，帮助用户直观了解体重变化情况。

#### 2.4.2 功能需求

| 需求项 | 详细说明 |
|--------|----------|
| 记录方式 | 手动输入体重数值 |
| 记录频率 | 每日一次或用户自定义周期 |
| 数据展示 | 体重变化曲线图（折线图） |
| 时间维度 | 支持查看：近7天、近30天、近90天、全部 |
| 目标线 | 显示目标体重参考线 |
| 数据统计 | 最高体重、最低体重、平均体重、减重总量 |

#### 2.4.3 数据字段

```
体重记录表：
- 记录ID
- 用户ID
- 记录日期
- 体重数值
- 备注（可选）
- 创建时间
```

#### 2.4.4 界面设计要点

- 首页快捷入口：今日体重记录
- 体重曲线图：横轴日期、纵轴体重
- 目标体重线：虚线标注
- 减重里程碑提醒：每减重5kg给予鼓励
- 支持数据导出（Excel格式）

#### 2.4.5 提醒功能

- 每日固定时间提醒记录体重（用户可设置提醒时间）
- 长时间未记录提醒（超过3天未记录）
- 达成阶段性目标提醒

### 2.5 随机推荐菜谱功能

#### 2.5.1 功能描述

为不想设置复杂偏好的用户提供快速随机推荐功能，一键获取推荐菜品或一日三餐。

#### 2.5.2 推荐模式

| 模式 | 说明 |
|------|------|
| 单顿推荐 | 随机推荐一道菜品（早餐/午餐/晚餐/加餐） |
| 一日三餐推荐 | 随机生成完整的一日三餐食谱 |

#### 2.5.3 功能特点

- 无需登录即可使用
- 无需填写健康信息
- 一键刷新，获取新推荐
- 支持简单筛选（荤/素、菜系）
- 推荐结果包含：菜品图片、食材、做法

#### 2.5.4 界面设计

```
随机推荐页面：
┌─────────────────────────┐
│    🎲 今天吃什么？      │
├─────────────────────────┤
│  [单顿推荐] [一日三餐]  │
├─────────────────────────┤
│  简单筛选：             │
│  ○ 荤菜 ○ 素菜 ○ 不限  │
│  ○ 川菜 ○ 粤菜 ○ 不限  │
├─────────────────────────┤
│      [🎲 换一个]        │
└─────────────────────────┘
```

## 五、技术架构概述

### 5.1 数据来源

| 数据类型 | 来源方式 |
|----------|----------|
| 食材价格数据 | 对接超市/市场价格API或爬虫采集 |

### 5.2 核心算法

| 算法模块 | 说明 |
|----------|------|
| 热量计算算法 | 基于BMR公式计算每日热量需求 |
